## 2024-05-24 - Fix Redundant TalkBack and Missing Action Labels
**Learning:** Found a common pattern in Compose components where `Icon` or `Image` elements had `contentDescription`s that matched accompanying `Text` labels (e.g., in `QuickTextButton` and `AppListItem`), causing redundant TalkBack announcements. Additionally, `clickable` modifiers were missing `onClickLabel`s, resulting in generic "Double tap to activate" announcements instead of descriptive actions.
**Action:** Always set `contentDescription = null` for visual elements that are immediately accompanied by descriptive text to reduce screen reader noise. Consistently provide an explicit `onClickLabel` in `clickable` modifiers so users know exactly what the action does (e.g., "Open" or the button text itself).
