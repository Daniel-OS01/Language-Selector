## 2024-05-17 - Avoid Intermediate Lists and String Allocations
**Learning:** Chaining `.sortedBy` calls in Kotlin creates intermediate lists, increasing memory pressure and processing time. Furthermore, using `.lowercase()` to perform case-insensitive comparisons inside filtering or sorting lambdas allocates new strings for every element, which can become a bottleneck when processing lists of applications or other data.
**Action:** Instead of chaining `.sortedBy`, use `.sortedWith(compareBy<Type> { primarySort }.thenBy(String.CASE_INSENSITIVE_ORDER) { secondarySort })`. Be mindful of the sort priority: reversing the property accessors is necessary to replicate the behavior of a chained `.sortedBy` because `.sortedBy` is stable. For searching and filtering strings, use `.contains(query, ignoreCase = true)` rather than allocating `.lowercase()` copies of strings.
