## 2023-11-09 - Kotlin String Case Insensitive Sorting and Filtering
**Learning:** In Kotlin (especially Jetpack Compose view models), doing `.lowercase()` in a loop inside `sortedBy` or `filter` generates an enormous amount of intermediate string allocations, negatively impacting performance. Additionally, chained `.sortedBy{}` calls create intermediate list copies under the hood.
**Action:** Always use `String.CASE_INSENSITIVE_ORDER` with `.sortedWith(compareBy(...).thenBy(...))` instead of chaining `sortedBy`. In string searching/filtering, utilize the `.contains("search", ignoreCase = true)` rather than doing `.lowercase()` allocations on large list elements. Explicitly define generic type parameter on `compareBy<Type>` to avoid compiler inference failures when chaining.
