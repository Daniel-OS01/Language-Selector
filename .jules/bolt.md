## 2026-04-10 - String Allocation Optimizations
**Learning:** In Kotlin list filtering/sorting inside Android ViewModels, chaining `.sortedBy` or continuously calling `.lowercase()` in loops creates measurable string allocation churn and garbage collection pressure, negatively impacting UI rendering loops. Combining chained sorts into a single `.sortedWith` and using `.contains(..., ignoreCase = true)` with `String.CASE_INSENSITIVE_ORDER` eliminates this intermediate list and string object creation overhead. Also, when suppressing MissingTranslation lint warnings via `tools:ignore`, ensure the root `<resources>` element in `strings.xml` includes the `xmlns:tools="http://schemas.android.com/tools"` namespace to prevent build failures.
**Action:** When sorting or filtering strings in ViewModels that power Compose UI, avoid chained sorts or inline `.lowercase()` mapping; default to `.sortedWith(compareBy(...).thenBy(...))` and `String.CASE_INSENSITIVE_ORDER`/ignoreCase extensions.
