## 2024-05-25 - Avoid String Allocations in Chained Sorts and Filters
**Learning:** In Kotlin (especially within ViewModels operating on collections of state), chaining `.sortedBy { it.property.lowercase() }` and `.filter { it.property.lowercase().contains(...) }` creates significant garbage collection overhead due to repeated intermediate string allocations for every item in the collection during render or filter loops.
**Action:** Replace `.lowercase().contains(...)` with `.contains(..., ignoreCase = true)`. Replace chained `.sortedBy {}` calls with `.sortedWith(compareBy<Type> { ... }.thenBy(String.CASE_INSENSITIVE_ORDER) { it.property })`. Ensure the `compareBy` parameters are provided in reverse order of the original `sortedBy` chain to maintain the same stable sort behavior.
