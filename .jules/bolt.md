## 2024-05-24 - Avoiding String Allocations in Kotlin List Filtering & Sorting
**Learning:** In Kotlin (especially within Android Jetpack Compose ViewModels), frequent updates triggered by rapid search inputs or list sorting can introduce significant memory overhead and subsequent GC pressure if new strings are allocated during the filtering process. This application's codebase previously exhibited this issue by calling `.lowercase()` on both query strings and list items inside `filter` and `sortedBy` operations. This caused excessive string allocation when comparing items. Also, multiple `.sortedBy` calls created unnecessary intermediate lists.
**Action:** When filtering strings, use `.contains(..., ignoreCase = true)` rather than creating lowercased copies. When chaining sorting conditions, use a single `.sortedWith(compareBy<Type> { ... }.thenBy(String.CASE_INSENSITIVE_ORDER) { ... })` instead of sequential `.sortedBy` calls. This optimizes memory allocation by preventing intermediate lists and extra string creations.
