## 2026-05-24 - Improve Compose Clickable Semantics and TalkBack Announcements
**Learning:** Generic Compose `clickable` modifiers without explicit roles or labels result in non-descriptive "Double tap to activate" TalkBack announcements. Additionally, explicitly defining `contentDescription` on icons within components that already announce state (like `FilterChip`) or have adjacent text causes redundant or confusing announcements.
**Action:** When adding interactivity in Jetpack Compose, utilize the `onClickLabel` parameter and a semantic `Role` (e.g., `Role.Button`) to clearly communicate the action. Ensure `contentDescription = null` for internal state-indicating or decorative icons where surrounding UI context or component defaults already provide sufficient meaning.
