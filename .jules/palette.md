## 2024-05-01 - Redundant TalkBack announcements and descriptive actions
**Learning:** In Jetpack Compose, an icon accompanied by text (e.g., inside a button or row) can cause redundant screen reader announcements. We should use `contentDescription = null` for the icon when its semantic meaning is fully covered by the adjacent text. Additionally, Compose's `clickable` and `combinedClickable` modifiers read a generic "Double tap to activate" announcement by default. Using `onClickLabel` and `onLongClickLabel` parameters allows providing contextually relevant descriptions (e.g., "Double tap to open").
**Action:** Always verify if an icon needs its own description or if adjacent text is sufficient, and set `contentDescription = null` when appropriate. When using `clickable` or `combinedClickable`, specify `onClickLabel` (and `onLongClickLabel`) to override generic TalkBack prompts with descriptive actions.
