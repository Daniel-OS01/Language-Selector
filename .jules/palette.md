## 2024-03-24 - Redundant TalkBack Announcements on State-Indicating Components
**Learning:** Components that inherently announce their state (such as `FilterChip` for selected state) cause redundant TalkBack announcements if their internal state-indicating icons (e.g., a checkmark) also have a `contentDescription`. Additionally, text buttons with visible text labels and accompanying icons cause repetitive announcements if both have descriptions.
**Action:** Always set `contentDescription = null` for internal state-indicating icons within stateful components (like chips or checkboxes), and for decorative icons in buttons where the text label already conveys the action. Use `onClickLabel` in `clickable` modifiers for clearer action descriptions (e.g., "Open" instead of default "Double tap to activate").
