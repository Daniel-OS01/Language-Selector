## 2024-06-11 - Jetpack Compose Accessibility: Redundant TalkBack announcements
**Learning:** TalkBack double-announces content if state-indicating components (like FilterChip checkmarks) or decorative images with adjacent text (like AppListItem app icon next to app name) have explicit `contentDescription` properties. Additionally, Jetpack Compose `clickable` modifiers default to a generic "Double tap to activate" announcement, which isn't descriptive.
**Action:** Use `contentDescription = null` for purely decorative icons or those paired with adjacent text/state indicators to prevent redundant TalkBack announcements. Always configure Jetpack Compose `clickable` modifiers with a specific semantic `role` (like `Role.Button`) and an `onClickLabel` to replace generic announcements with descriptive actions.
