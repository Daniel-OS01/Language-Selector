## 2024-05-14 - Redundant screen reader announcements in FilterChip and purely decorative AppListItem icons
**Learning:** For accessibility in Android Jetpack Compose, icons within components that inherently manage their own accessibility state (such as the 'Done' icon in a selected FilterChip) should have `contentDescription = null` to prevent redundant TalkBack announcements. Additionally, purely decorative icons or those accompanied by adjacent descriptive text (e.g., an app icon next to the app name in a list item) should also have `contentDescription = null`.
**Action:** When working with Material3 components like `FilterChip` that include stateful icons, or purely decorative imagery accompanied by descriptive text, always set `contentDescription = null` unless the icon itself provides unique, non-redundant information. For functional icons like back buttons or more options, ensure they use descriptive, localized string resources rather than hardcoded english text.
