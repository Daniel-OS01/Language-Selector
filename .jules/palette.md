## 2026-07-22 - Fix redundant TalkBack announcements
**Learning:** In Jetpack Compose, icons within components that inherently manage their own accessibility state (like the 'Done' icon in a selected FilterChip), or decorative icons adjacent to text labels (like app icons in list items), cause redundant TalkBack announcements if they have a contentDescription. Furthermore, clickable Rows acting as list items need a semantic Role (like Role.Button) so screen readers announce them properly as interactive elements.
**Action:** Set contentDescription to null for decorative/redundant icons inside existing accessible components or adjacent to primary text labels. Explicitly set semantic roles on custom clickable layout primitives (like Row/Column) to ensure correct screen reader announcements.
