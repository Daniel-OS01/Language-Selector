## 2024-05-18 - Accessibility: Redundant announcements for internal state icons
**Learning:** In Jetpack Compose, icons within components that inherently manage their own accessibility state (such as the 'Done' icon in a selected `FilterChip`, or purely decorative icons adjacent to text labels like in app list items) often result in redundant TalkBack announcements if they have an explicit `contentDescription` set. The parent component (`FilterChip` or `ListItem` wrapper) typically announces the state or primary label itself.
**Action:** Set `contentDescription = null` for internal state-indicating icons (e.g., `FilterChip` leading icons) and adjacent decorative icons to prevent double-announcing states or labels that are already handled by the parent component or sibling text elements.
