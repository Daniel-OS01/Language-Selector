## 2024-06-23 - Removed contentDescription from redundant accessibility elements
**Learning:** Found multiple instances where Jetpack Compose clickable areas and components that already have semantic meaning/labels had redundant `contentDescription` on purely decorative child icons. Adding redundant string labels to purely decorative icons (or icons grouped directly alongside descriptive text, like app titles and labels inside `FilterChip`) triggers duplicate, confusing screen reader announcements.
**Action:** Applied semantic `role = Role.Button` to `clickable` and `combinedClickable` containers, and set `contentDescription = null` for all child `Icon`/`Image` elements inherently explained by their adjacent text or parent context. Next time encountering Compose icon groups, explicitly check if the outer wrapper provides semantics to nullify inner element descriptions to avoid verbosity in TalkBack.
