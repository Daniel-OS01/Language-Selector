## 2024-06-04 - Prevent Unauthorized Data Extraction via Backup
**Vulnerability:** The Android app allowed app data backups (`android:allowBackup="true"`) and did not explicitly exclude sensitive domains from extraction rules. This could allow unauthorized access to sensitive application data (like databases, shared preferences, etc.) via local backups or device-to-device transfers.
**Learning:** For apps dealing with sensitive or system-level configurations (like this one using Shizuku), allowing broad data backups is risky. On Android 12+ (API 31+), `android:allowBackup="false"` is ignored for device-to-device transfers, so explicit `<exclude>` rules in `data_extraction_rules.xml` are mandatory to prevent data extraction. Each `<exclude>` element must define both `domain` and `path` to avoid XML syntax errors.
**Prevention:** Set `android:allowBackup="false"` in `AndroidManifest.xml` and define explicit `<exclude>` entries for all domains (`root`, `file`, `database`, `sharedpref`, `external`) in both `data_extraction_rules.xml` (`<cloud-backup>` and `<device-transfer>`) and `backup_rules.xml` (`<full-backup-content>`).
