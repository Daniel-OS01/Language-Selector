## 2024-05-18 - Securely Disable App Backups on Android 12+
**Vulnerability:** Android backups were enabled via `android:allowBackup="true"`, and the data extraction/backup rules were configured to include everything by default. This allowed potential extraction of sensitive app data using ADB or cloud backups.
**Learning:** Setting `android:allowBackup="false"` in `AndroidManifest.xml` is insufficient to prevent all forms of data extraction, particularly device-to-device transfers on Android 12+ (API 31+). The `data_extraction_rules.xml` (for Android 12+) and `backup_rules.xml` (for older versions) must explicitly exclude all data domains to ensure sensitive information cannot be exported.
**Prevention:** To securely prevent data extraction on modern Android versions, set `android:allowBackup="false"` AND explicitly define `<exclude domain="..." path="." />` for `root`, `file`, `database`, `sharedpref`, and `external` within `<cloud-backup>`, `<device-transfer>`, and `<full-backup-content>` configurations.
