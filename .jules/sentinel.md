## 2024-05-20 - Securely Disabling Backups on Android 12+
**Vulnerability:** Application backups were enabled (`android:allowBackup="true"`) and no domains were explicitly excluded in extraction or backup rules. This could allow for sensitive data extraction via backups or device transfers.
**Learning:** To securely disable backups on Android 12+ (API 31+), simply setting `android:allowBackup="false"` in the Manifest is insufficient, because it is ignored for device-to-device transfers. You must retain `android:dataExtractionRules` and explicitly `<exclude>` all domains (root, file, database, sharedpref, external) under both `<cloud-backup>` and `<device-transfer>` in `data_extraction_rules.xml`. The same explicitly named `<exclude>` rules must also be applied under `<full-backup-content>` in `backup_rules.xml`.
**Prevention:** Every time backups need to be disabled, explicitly exclude all domains in both XML rule files rather than just changing the manifest flag, to ensure full coverage against sensitive data extraction during backups or device transfers.
