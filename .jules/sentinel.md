## 2024-05-18 - Prevent sensitive data extraction via Android backup mechanisms
**Vulnerability:** Application backups could inadvertently expose sensitive user data, particularly over device-to-device transfers, because `android:allowBackup="true"` was enabled and explicit exclusions were not defined in `data_extraction_rules.xml` and `backup_rules.xml`.
**Learning:** To securely disable backups on Android 12+ (API 31+), setting `android:allowBackup="false"` is insufficient since it's ignored for device-to-device transfers. Explicit `<exclude>` tags with correct `domain` and `path` must be defined under `<cloud-backup>` and `<device-transfer>` inside `data_extraction_rules.xml`, and identically configured in `backup_rules.xml` under `<full-backup-content>`.
**Prevention:** In Android applications containing or producing sensitive data, always explicitly disable backups or provide strict exclusions across all backup mechanisms (`cloud-backup`, `device-transfer`, `full-backup-content`) during initial application configuration.
