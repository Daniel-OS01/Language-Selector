## 2025-01-28 - Insecure Backup Configuration
**Vulnerability:** The application was configured to allow automatic backups (`android:allowBackup="true"`) without any restrictions. For devices running Android 12+ (API 31+), `android:allowBackup="false"` is ignored for device-to-device transfers, potentially leading to sensitive data extraction during transfers.
**Learning:** To securely and fully disable all backups and device-to-device data extraction on Android 12+, we must retain `android:dataExtractionRules` and explicitly `<exclude>` all domains (`root`, `file`, `database`, `sharedpref`, `external`) under both `<cloud-backup>` and `<device-transfer>` in `data_extraction_rules.xml`. We must also do the same in `backup_rules.xml` under `<full-backup-content>`. Furthermore, every `<exclude>` or `<include>` element must explicitly declare both the `domain` and `path` attributes (e.g., `<exclude domain="root" path="." />`) to prevent Android XML syntax errors.
**Prevention:** Ensure `android:allowBackup="false"` is explicitly set, and configure `data_extraction_rules.xml` and `backup_rules.xml` to explicitly exclude all directories/domains. Test that there are no XML syntax errors.
