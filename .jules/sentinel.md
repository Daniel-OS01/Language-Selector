## 2024-05-15 - Securely Disabling Backups on Android 12+
**Vulnerability:** Application data could be extracted via device-to-device transfer or cloud backup mechanisms, even if `android:allowBackup="false"` was set, due to Android 12+ changes in `data_extraction_rules.xml`.
**Learning:** Setting `android:allowBackup="false"` in `AndroidManifest.xml` is insufficient to prevent data extraction via device transfers on API 31+. The system ignores it for device-to-device transfers. To securely prevent backup and transfer, explicit `<exclude>` rules must be defined for all domains (`root`, `file`, `database`, `sharedpref`, `external`) under both `<cloud-backup>` and `<device-transfer>` in `data_extraction_rules.xml`, and similarly in `<full-backup-content>` within `backup_rules.xml`.
**Prevention:** Always define explicit exclusion rules in both `data_extraction_rules.xml` and `backup_rules.xml` when `android:allowBackup="false"` is required, ensuring all storage domains are covered.
