## 2024-05-24 - Securely Disable Android Backups and Device Transfers
**Vulnerability:** Application allows sensitive data to be backed up to the cloud or transferred device-to-device because `android:allowBackup="true"` is enabled. Even when it is set to `false`, it's not sufficient to prevent device-to-device transfers on Android 12+ (API 31+).
**Learning:** To securely disable data extraction on Android 12+ (API 31+), setting `android:allowBackup="false"` in the Manifest is ignored for device-to-device transfers. You must retain `android:dataExtractionRules` and explicitly `<exclude>` all domains (root, file, database, sharedpref, external) under both `<cloud-backup>` and `<device-transfer>` in `data_extraction_rules.xml`.
**Prevention:** Always apply the explicit `<exclude>` rules under `<cloud-backup>` and `<device-transfer>` in `data_extraction_rules.xml`, and apply the same `<exclude>` rules under `<full-backup-content>` in `backup_rules.xml` when the application's intention is to disallow all backups or transfers. Every `<exclude>` or `<include>` element must explicitly declare both the `domain` and `path` attributes.
