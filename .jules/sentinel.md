## 2024-06-11 - Insufficient Application Backup Disabling
**Vulnerability:** Application backups could not be fully disabled on Android 12+ (API 31+). Even with `android:allowBackup="false"` in `AndroidManifest.xml`, backups for device-to-device transfers could still be implicitly enabled because `android:dataExtractionRules` and `android:fullBackupContent` were left to the system's default behavior or had commented-out rules.
**Learning:** To securely and completely disable all forms of application backups and data extraction on Android 12+ devices, you must not only set `android:allowBackup="false"` in `AndroidManifest.xml` but also explicitly configure the XML resources for `dataExtractionRules` and `fullBackupContent`. Every backup mode (cloud-backup, device-transfer, and full-backup-content) must explicitly use `<exclude>` tags specifying both `domain` and `path` for all storage domains (root, file, database, sharedpref, external).
**Prevention:** When dealing with sensitive apps or intending to completely disable backups, ensure `android:allowBackup="false"` is accompanied by explicitly configured backup XML rules that exclude all domains. Never rely solely on the `allowBackup` flag or the system default backup behavior for Android 12+.
