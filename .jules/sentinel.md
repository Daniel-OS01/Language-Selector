## 2024-05-18 - Prevent Sensitive Data Extraction via Backup
**Vulnerability:** The application allowed data extraction via device-to-device transfers and cloud backups, even if `android:allowBackup="false"` was set, due to missing explicit exclusions in `data_extraction_rules.xml` and `backup_rules.xml`.
**Learning:** On Android 12+ (API 31+), setting `android:allowBackup="false"` is insufficient because it is ignored for device-to-device transfers. You must retain `android:dataExtractionRules` and explicitly `<exclude>` all domains (root, file, database, sharedpref, external) under both `<cloud-backup>` and `<device-transfer>` in `data_extraction_rules.xml`, and apply the same `<exclude>` rules under `<full-backup-content>` in `backup_rules.xml`. Every `<exclude>` or `<include>` element must explicitly declare both the `domain` and `path` attributes (e.g., `<exclude domain="root" path="." />`) to avoid Android XML syntax errors.
**Prevention:** Always explicitly exclude all domains in both data extraction and backup rules XML files when attempting to secure an app against data extraction via backups or device transfers.
