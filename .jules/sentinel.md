## 2024-05-24 - Device-to-Device Backup Extraction
**Vulnerability:** The application was allowing arbitrary external access/extraction to application data via device-to-device transfers (D2D). Setting `android:allowBackup="true"` and leaving the data extraction rules unconfigured exposed data.
**Learning:** `android:allowBackup="false"` is insufficient to stop data extraction in Android 12+ (API 31+). The flag is ignored for D2D transfers. You must retain `android:dataExtractionRules` and explicitly `<exclude>` all domains (`root`, `file`, `database`, `sharedpref`, `external`) under both `<cloud-backup>` and `<device-transfer>` in `data_extraction_rules.xml` and under `<full-backup-content>` in `backup_rules.xml` (for API < 31 devices). Every exclusion must declare both `domain` and `path`.
**Prevention:** To thoroughly disable data backup/extraction functionality, you must configure both XML files explicitly excluding all file domains along with setting `android:allowBackup="false"`.
